# Milvus Infrastructure

Milvus 2.5.6 vector database for semantic search and vector storage.

## Overview

This module provides Milvus infrastructure for the Insurance Claims Processing System. Milvus serves as the vector database for storing and searching document embeddings generated by the RAG service using Google Gemini embeddings.

## Features

- High-performance vector similarity search
- HNSW indexing for fast approximate nearest neighbor search
- COSINE similarity metric for semantic search
- Object storage with MinIO for persistence
- Etcd for metadata management
- Attu web UI for database management

## Quick Start

Start Milvus and all dependencies:

```bash
docker-compose -f infrastructure/milvus/docker-compose.yml up -d
```

Or start via main compose file:

```bash
docker-compose up -d milvus etcd minio attu
```

Wait for services to be ready (approximately 30 seconds).

## Collection Schema

### insurance_kb_v2

The main collection for storing document embeddings.

```python
{
    "id": VARCHAR (primary key),
    "content": VARCHAR (text content),
    "embedding": FLOAT_VECTOR (3072 dimensions),
    "metadata": JSON (additional info),
    "doc_type": VARCHAR (document type),
    "parent_id": VARCHAR (parent chunk id)
}
```

### Index Configuration

- **Type**: HNSW (Hierarchical Navigable Small World)
- **Metric**: COSINE
- **M**: 16
- **efConstruction**: 200

## Usage

### Attu Web UI

Access the web UI at: http://localhost:8000

Features:
- View collections and partitions
- Query and search vectors
- Monitor system metrics
- Manage indexes

### MinIO Console

Access the console at: http://localhost:9001

Default credentials:
- Username: `minioadmin`
- Password: `minioadmin`

### Python Client

```python
from pymilvus import connections, Collection

# Connect to Milvus
connections.connect(host="localhost", port=19530)

# Get collection
collection = Collection("insurance_kb_v2")

# Search
results = collection.search(
    data=[query_embedding],
    anns_field="embedding",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=5
)
```


## Troubleshooting

### Connection refused

- Verify all containers are running: `docker-compose ps`
- Check port 19530 is not in use by another process
- Review logs: `docker-compose logs milvus`

### Collection creation fails

- Verify embedding dimensions match (3072 for Gemini)
- Check that MinIO is accessible for storage
- Review Milvus logs for specific errors

### Search performance issues

- Verify HNSW index is built: `collection.load()`
- Check `ef` parameter is appropriate (higher = more accurate, slower)
- Monitor system resources (CPU, memory)

### Attu not accessible

- Verify Attu container is running
- Check port 8000 is not in use
- Ensure Milvus is fully started before Attu

### Data persistence issues

- Verify MinIO is running and accessible
- Check disk space on host machine
- Review MinIO logs: `docker-compose logs minio`
